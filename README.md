# 계산기 만들기

<br>

## 계산기 기능

> * 사칙연산을 수행할수 있습니다.
> * exit를 입력하기 전까지 계산을 반복해서 수행합니다.
> * 계산기록을 확인할 수 있습니다.
> * 계산기록을 삭제할 수 있습니다.
> * 계산기록을 입력값과 비교해서 확인할 수 있습니다.
> * 나눗셈 연산 시 분모에 0이 입력되는 오류를 방지합니다.

## 🛠️ 사용 기술 (Tech Stack)

* **Language:** Java (JDK 17 이상 권장)
* **IDE:** IntelliJ IDEA

## 🚨 Trouble Shooting

### 1. 계산 기록 출력의 가독성 문제 
* **문제 상황:** 처음에는 계산 결과값만 저장하기 위해 `private ArrayList<Integer> calList`로 선언함. 하지만 기록을 호출해 보니 `453`, `234`처럼 숫자만 덩그러니 출력되어 어떤 수식의 결과인지 파악하기 매우 어려웠음.
* **해결 방법:** 리스트의 제네릭 타입을 `ArrayList<String>`으로 수정함. 연산 수행 후 `calList.add(a + " + " + b + " = " + result)`와 같이 수식 전체를 하나의 문자열로 묶어서 저장하도록 변경함.

### 2. 제어문 꼬임 및 향상된 for문 문법 오류
* **문제 상황:** 초기에 `if-else`문으로 모든 메뉴 조건(`exit`, 사칙연산, `record`, `remove`)을 처리하려다 보니 로직의 실행 순서가 꼬임. 또한, 향상된 `for`문을 사용해 계산 기록을 한 줄씩 출력하려다 문법 에러가 발생함.
* **해결 방법:** 사용자의 입력 명령어(메뉴)를 기준으로 `switch`문을 도입하여 로직 분기를 깔끔하게 나눔. 향상된 `for`문은 괄호 구조를 정확히 맞추어 `for (String history : calList)` 형태로 수정하여 데이터를 정상적으로 출력함.

### 3. Enum 구조 설계의 어려움
* **문제 상황:** Enum을 처음 도입할 때, 실제 연산 기호(`+`, `-`)를 상수(`SUM`, `SUB`)에 어떻게 매핑하고 꺼내 써야 하는지 문법적으로 헤매어 초기 구현에 시간이 걸림.
* **해결 방법:** Enum 내부에 `private final String type;` 필드를 선언하고, 생성자를 통해 값을 연결해 준 뒤 `getType()` 메서드를 만들어 실제 문자열을 꺼내올 수 있도록 해결함.

### 4. Enum의 내부 구조 설계 및 활용
* **문제 상황:** `OperatorType` Enum을 생성했으나, 실제 연산 기호(`+`, `-`)를 `SUM`, `SUB` 같은 상수에 어떻게 매핑하고 꺼내 써야 하는지 문법적으로 헷갈려 초기 구현에 크게 헤맸음.
* **해결 방법:** Enum 내부도 일반 클래스처럼 구성할 수 있음을 복습함. `private final String type;` 필드를 선언하고, 생성자를 통해 `SUM("+")`처럼 값을 연결함. 이후 `getType()` Getter 메서드를 만들어 실제 연산 기호를 꺼내오도록 구현함.

### 5. 제네릭(Generics) 타입의 사칙연산 에러
* **문제 상황:** 제네릭으로 받은 매개변수 `T a`, `T b`를 연산 로직에 바로 넣었더니 에러가 발생함. (자바는 `T`가 숫자인지 문자인지 모르기 때문에 연산자를 사용할 수 없음)
* **해결 방법:** `a.doubleValue()`와 `b.doubleValue()` 메서드를 사용하여 값을 변환한 뒤 연산을 수행함.

### 6. 제네릭(Generics) 원시 타입 경고
* **문제 상황:** `ArithmeticCalculator cal = new ArithmeticCalculator();` 생성 시 제네릭 타입을 명시하지 않아 '원시 사용(Raw Use)' 및 '확인되지 않은 대입' 경고가 발생함.
* **해결 방법:** 객체 생성 시 꺾쇠 안에 `<Double>` 등 명확한 데이터 타입을 선언하여 컴파일러에게 타입 정보를 제공함으로써 경고를 해결하고 안전성을 확보함.

### 7. 반환 타입(Return Type) 불일치 에러
* **문제 상황:** 제네릭을 적용한 후 연산 결과를 반환할 때 호환되지 않는 타입(`Incompatible types`) 에러가 발생함.
* **해결 방법:** `calculate` 메서드의 선언부를 `public int calculate`에서 `public double calculate`로 변경함.

### 8. 함수형 인터페이스(BiFunction) 호출 규칙
* **문제 상황:** `BiFunction`을 사용한 Enum 객체에서 연산을 실행할 때, 임의로 `oper.cal(a, b)`라고 호출했더니 존재하지 않는 메서드라며 에러가 발생함.
* **해결 방법:** 메서드 호출부를 `oper.apply(a, b)`로 수정하여 정상 작동시킴.
